# Setup Guide: Google Sheets API Authentication

This guide explains how to configure Google Sheets API access for the Budgeting Automation tool using a **Service Account**.

---

## Step 1: Create a Google Cloud Project & Enable APIs

1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. Log in with your Google account.
3. Click the project dropdown in the top navigation bar and select **New Project**.
4. Give your project a name (e.g., `Personal-Budget-Automation`) and click **Create**.
5. Once created, select the project.
6. Enable the **Google Sheets API**:
   - Go to **APIs & Services** > **Library** in the left sidebar.
   - Search for **Google Sheets API**.
   - Click on it and click **Enable**.
7. Enable the **Google Drive API**:
   - Return to **APIs & Services** > **Library**.
   - Search for **Google Drive API**.
   - Click on it and click **Enable**.

---

## Step 2: Create a Service Account and Download Key File

1. In the Google Cloud Console, navigate to **IAM & Admin** > **Service Accounts** in the left sidebar.
2. Click **Create Service Account** at the top.
3. Enter a name (e.g., `budget-uploader`) and click **Create and Continue**.
4. (Optional) Skip assigning roles, and click **Done**.
5. You will see your new service account in the list. Copy the email address of the service account (it will look like `budget-uploader@your-project-id.iam.gserviceaccount.com`). You will need this in Step 3.
6. Click on the service account's email address, then navigate to the **Keys** tab at the top.
7. Click **Add Key** > **Create new key**.
8. Select **JSON** as the key type and click **Create**.
9. The JSON file will download automatically to your computer.
10. Rename this file to `service_account.json` and move it to your project directory:
    `/Users/angel/Documents/Dev/Budgeting/service_account.json`

---

## Step 3: Share the Google Sheet

1. Open your browser and open your Google Sheet named **Budgeting Sheet**.
2. Click the **Share** button in the top-right corner.
3. Paste the Service Account email address you copied in Step 2.
4. Set the permissions role to **Editor** (uncheck "Notify people" to avoid email errors since it's a service account).
5. Click **Share** / **Save**.

## Step 4: Set Up Virtual Environment & Run

1. Open a terminal and navigate to the project directory:
   ```bash
   cd /Users/angel/Documents/Dev/Budgeting
   ```

2. Create a Python virtual environment:
   ```bash
   python3 -m venv venv
   ```

3. Activate the virtual environment:
   - On macOS/Linux:
     ```bash
     source venv/bin/activate
     ```

4. Install the required Python packages inside the virtual environment:
   ```bash
   pip install -r requirements.txt
   ```

5. Run the automation script:
   ```bash
   python app.py
   ```

> [!TIP]
> Run the script with `--dry-run` to see what changes would be made without actually modifying the Google Sheet:
> ```bash
> python app.py --dry-run
> ```

6. When you are finished, you can deactivate the virtual environment:
   ```bash
   deactivate
   ```

## Step 5: Secure Your Credentials in macOS Keychain (Recommended)

To avoid storing your sensitive `service_account.json` file in plain text on your hard drive, you can load its contents directly into the secure macOS Keychain using the Python `keyring` package.

1. Activate your virtual environment and make sure requirements are installed:
   ```bash
   source venv/bin/activate
   pip install -r requirements.txt
   ```

2. Run the following command. It will prompt you for input; copy the full contents of your `service_account.json` file (including the outer `{ }`), paste it in the prompt, and press Enter:
   ```bash
   python3 -c "import keyring; keyring.set_password('BudgetingAutomation', 'google_service_account', input('Paste JSON key contents: '))"
   ```

3. Once saved, you can safely delete the local plain-text file from disk:
   ```bash
   rm service_account.json
   ```

The script will now automatically fetch and use the credentials from your Keychain. If you ever need to update or replace the credentials, simply re-run the Python command in step 2.
